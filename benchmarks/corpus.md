# Corpus statistics (Phase 1)

> **Claims hygiene:** every table here is regenerated by committed scripts, never
> written from memory. The **reference corpus** numbers require downloading the
> PubMed baseline (a deliberate, networked step the user runs); until then they are
> marked pending. The **pipeline demonstration** numbers below are the real output
> of the committed sample fixture and reproduce exactly.

## How to regenerate

```bash
# 1. Download the domain-filtered PubMed baseline (network; see scripts/ingest.py)
#    then build the store and the reference table:
uv run python scripts/ingest.py data/baseline --db data/corpus.sqlite \
    --report benchmarks/corpus.md

# 2. Train the tokenizer and append the fertility sweep (ADR-0003):
uv run python scripts/train_tokenizer.py --db data/corpus.sqlite \
    --general data/msmarco_sample.txt --out artifacts/tokenizer.json \
    --vocab-sizes 16000 32000 --report benchmarks/corpus.md
```

### Tokenizer fertility sweep on the sample (illustrative)

`scripts/train_tokenizer.py` on the sample store with a 4-line general corpus and
